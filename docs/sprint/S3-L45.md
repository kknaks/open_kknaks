## 구현 결정사항

1. **result()/stream() Python 3.10 호환**: `asyncio.timeout` (3.11+) 대신 `asyncio.wait_for()` 사용.

2. **Client.stream()**: 내부 async generator를 한 번 감싸서 yield. 직접 `asyncio.wait_for`로 전체 generator를 감싸면 부분 소비가 안 되므로 이벤트별 yield 방식 유지.

3. **BatchRunner.wait_batch()**: subscribe_chunks 대신 polling 방식. 배치의 개별 task 스트림을 모두 구독하는 것은 복잡도 대비 이점이 적음. poll_interval=1.0으로 충분.

4. **CLI 구조**: 각 sub-command 파일에서 typer.Typer() 생성 → main.py에서 add_typer(). 지연 import으로 optional deps(redis) 로딩 최적화.

5. **MCP server**: `create_server()` 팩토리 함수 → Server 인스턴스 반환. mcp 라이브러리의 untyped decorator에 `type: ignore` 사용.

6. **__init__.py**: `__all__` isort-style 알파벳 정렬 (ruff RUF022).

## 발견된 이슈

- MCP 라이브러리 (`mcp>=1.6`)의 `@server.list_tools()`, `@server.call_tool()` 데코레이터가 untyped — mypy strict에서 `type: ignore` 필요.
- `asyncio.timeout`은 Python 3.11+이므로 3.10 타겟에서 사용 불가.

## 인터페이스 변경 이력

- BatchRunner: `batch_id`를 Redis에 저장하지 않고 클라이언트 측에서 관리. `wait_batch(task_ids=...)` 형태로 task_id 리스트를 직접 전달.
- MCP: `stream_task` 도구 제외 — MCP tool call은 동기적 응답이므로 스트리밍 부적합.
