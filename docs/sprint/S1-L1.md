## 구현 결정사항

- PTYProcess: 클래스 기반 (dataclass 아님) — `_closed` 상태 관리 필요
- PTYProcess: `_close_fd()` 멱등성 보장 — `_closed` 플래그로 이중 close 방지
- StreamParser: `strip_ansi()` 공개 함수로 분리 — 다른 곳에서도 재사용 가능
- StreamParser: `parse_stream_json_line()` 반환 타입 `dict[str, Any]` — 동적 파싱 결과
- AbstractBroker: `subscribe_chunks()`는 sync `def` (AsyncIterator 반환만)
- AbstractBroker: middleware 시그널 (emit_before/emit_after) 제외 — Worker에서 직접 처리

## 발견된 이슈

- `os.waitpid(-pgid, WNOHANG)` 패턴은 구현 안 함 → PTYProcess._reap()에서 self.pid만 waitpid
  - 이유: 프로세스 그룹 전체 reap은 Worker 레벨에서 관리하는 게 적절

## 인터페이스 변경 이력

- AbstractBroker에서 `emit_before`/`emit_after` 시그널 메서드 제거
  - 이유: Middleware 체인 실행은 Worker._process_task()에서 직접 수행 (Dramatiq 패턴)
- `subscribe_chunks()` sync def로 변경 (async def → def) — AsyncIterator 팩토리

## 다음 레이어(L2)에 전달할 주의사항
